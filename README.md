# high_school

高中学习笔记仓库。这里用于沉淀各学科的学习产出、章节笔记、专题总结、题库、错题和可复习成品。

本仓库只保存整理后的学习资料，不保存教材 PDF、课程标准 PDF、讲义原件、试卷原卷等原始资料。原始资料统一放在外层目录中维护，例如 `../教材/`、`../课程标准/`、`../原始资料/`。

## 仓库定位

这个仓库的目标是把高中学习内容整理成可长期复习、可持续补充、可提交到 GitHub 的 Markdown 知识库。

适合放入本仓库的内容：

- 按教材章节整理的核心知识点讲解。
- 跨章节专题总结。
- 解题方法、答题模板和学习方法。
- 题库、试卷解析和错题复盘。
- 可打印、可分享、可发布的复习成品。
- 笔记中需要引用的轻量图片素材。

不应放入本仓库的内容：

- 教材 PDF。
- 课程标准 PDF。
- 讲义原件、试卷原卷、图册、参考资料原件。
- 临时转换文件、脚本缓存、大体积无关文件。

## 栏目说明

### `学习产出/`

存放阶段性学习成果、综合整理、课程化输出和较完整的学习记录。

适合内容：

- 阶段复盘。
- 单元学习总结。
- 一段时间学习后的综合输出。

### `章节笔记/`

存放按教材章节组织的核心知识点讲解。

结构要求：

```text
物理/
  章节笔记/
    第一章 运动的描述/
      1.1 质点 参考系.md
      1.2 时间 位移.md
```

规则：

- 一个文件夹对应一个教材章节。
- 一个 Markdown 文件对应章节下的一个子章节。
- 文件名使用 `小节序号 小节名称.md`。
- 内容应通俗、完整、适合自学和复习。

每个子章节文件建议包含：

- 本节学习目标
- 核心知识点讲解
- 重点梳理
- 难点突破
- 例题讲解
- 易错点整理
- 考点考证点整理
- 练习题
- 练习题答案

### `知识点专题/`

存放跨章节、跨小节的专题整理，重点是把同一类概念、模型、规律串起来。

例子：

- 物理：运动图像专题、受力分析专题、牛顿运动定律应用专题。
- 数学：函数单调性专题、三角恒等变换专题、数列求和专题。
- 化学：离子反应专题、氧化还原专题、化学平衡专题。

### `方法总结/`

存放可迁移的学习方法、解题流程、审题方法和答题模板。

例子：

- 受力分析四步法。
- 运动学题目建模流程。
- 阅读理解定位法。
- 作文审题立意方法。

### `题库整理/`

存放按章节、知识点、题型、难度或考试来源整理的题目集合。

要求：

- 题目应标注知识点和难度。
- 题目与答案解析分开。
- 解析要说明思路，不只给最终结果。
- 错题应进一步沉淀到 `错题本/`。

### `试卷解析/`

存放整张试卷、单元测试、月考、期中、期末、模拟卷的讲评和复盘。

重点记录：

- 试卷信息。
- 知识点分布。
- 逐题解析。
- 错题分类。
- 失分原因。
- 后续复习计划。

### `错题本/`

存放个人真实错题、反复错题、易混题和需要二次订正的题。

每道错题建议记录：

- 来源和日期。
- 知识点。
- 错因类型。
- 原题。
- 错误思路。
- 正确解法。
- 防错提醒。
- 二次订正记录。

### `图片素材/`

存放学习笔记中需要引用的图片素材，例如示意图、函数图像、实验装置图、知识结构图。

要求：

- 图片文件名要能说明学科、章节、知识点和图片内容。
- 使用相对路径引用图片。
- 不保存教材背景图、整页截图或无学习价值的装饰图。
- 尽量压缩图片体积。

### `输出成品/`

存放已经整理成熟、可打印、可分享或可发布的内容。

例子：

- 单元复习讲义。
- 考前速查手册。
- 一页纸知识地图。
- 专题讲义。
- 可打印练习卷。

## 原始资料与转换流程

本仓库外层有资料管理目录：

```text
../教材/
../课程标准/
../原始资料/
../markdown/
```

推荐流程：

1. 教材 PDF 放在 `../教材/学科/`。
2. 课程标准放在 `../课程标准/`。
3. 讲义、试卷原卷、图册等放在 `../原始资料/学科/`。
4. PDF 转 Markdown 的中间文件放在 `../markdown/`。
5. 格式化、拆分、整理完成后，再将真正的学习笔记放入本仓库对应学科目录。

## 写作要求

- 面向高中生，语言要通俗清楚。
- 先讲直观理解，再讲概念定义、公式、适用条件和题目应用。
- 公式要说明符号含义、单位、适用条件和常见变形。
- 例题讲解要完整，不跳关键步骤。
- 易错点要说明“为什么错”和“如何避免”。
- 练习题要有答案和必要解析。
- 避免简单照搬教材原文，应整理成适合复习和自学的内容。

## Git 规则

- 空目录使用 `.gitkeep` 占位。
- 不提交 PDF、原始资料、大体积无关文件。
- 新增笔记前先确认学科归属，再放入对应学科目录。
- 提交前检查是否误加入原始资料或临时文件。

更详细的目录规范、章节笔记模板和各类学习产出要求，请参考外层 [AGENTS.md](../AGENTS.md)。
